Inference modes
Three modes. One pipeline.
NeuroDetect AI automatically selects the best inference strategy based on input format — no configuration needed at the point of care.
Mode 1 — Best accuracy

DICOM / Hounsfield units

Uses real clinical HU values from embedded DICOM metadata for maximum precision.

  • Apply rescale slope/intercept → true HU values
  • Brain window: WL 35, WW 100 (standard CT)
  • Skull strip via connected-component analysis
  • Hypodense band: HU 15–35 (below parenchyma)
  • Morphological open/close + remove <50 px regions
Mode 2 — Works anywhere

JPG / PNG adaptive threshold

Operates on standard images without HU metadata — suitable for digitised films.

  • CLAHE contrast enhancement
  • Approximate skull stripping
  • Compute tissue median and MAD inside brain
  • Flag pixels >1.5× MAD below median as hypodense
  • Morphological cleanup + remove <30 px regions
Mode 3 — Trained model

2.5D U-Net checkpoint

Highest accuracy when trained weights are provided — the full deep-learning pathway.

  • 2.5D U-Net with 5-channel pseudo-volume input
  • Trained on NIfTI volumes (pydicom → nibabel)
  • Auto-activates when .pth checkpoint is supplied
  • Encoder–decoder with skip connections
  • Volumetric batch inference via CLI or web app
Features
Built for clinical radiology workflows

Hypodense detection

Identifies regions darker than surrounding parenchyma — ischaemic stroke, cerebral oedema, and infarction — using validated HU thresholding or trained U-Net inference.

AI overlay visualisation

Semi-transparent red fill with orange contour outlines on detected lesions. Rendered side-by-side with original scan and binary segmentation mask for fast comparison.

Explainable AI (XAI)

Plain-language clinical interpretation of findings — region, lesion load %, HU range, and differential. Radiologist-auditable for clinical governance requirements.

Structured report generation

Auto-generated radiology-style text reports via report_generator.py with lesion load metrics, volumetric analysis, and ICD-10 suggestion.

Multi-format ingestion

Accepts DICOM (.dcm) with full HU metadata for maximum accuracy, or standard JPG/PNG via adaptive intensity thresholding. UUID-prefixed filenames prevent upload collisions.

PACS / RIS integration

Push results directly to hospital PACS via HL7 FHIR or DICOM SR. Drag-and-drop web uploader with AJAX inference or batch CLI for overnight processing jobs.
